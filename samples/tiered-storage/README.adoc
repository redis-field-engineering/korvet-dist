= Korvet Tiered Storage Sample
:toc:
:icons: font

== Overview

This sample demonstrates Korvet's tiered storage feature with:

* **Local (hot) tier**: Redis Enterprise Flex database on `localhost:12000`
* **Remote (cold) tier**: an Apache Iceberg table on MinIO (S3-compatible local storage)
* **Segment duration**: 10 seconds (fast rollover for demo purposes)

Data flows from the local tier to the remote tier as segments are sealed and offloaded, enabling cost-effective storage while maintaining Kafka API compatibility.

== Prerequisites

* Docker 20.10+
* Redis Enterprise Flex database running on `localhost:12000`

== Quick Start

[source,bash]
----
cd tiered-storage
make up           # Start Korvet + MinIO
make test-tiered  # Run tiered storage test
make down         # Stop services
----

== How It Works

=== Segments and the Storage Worker

Korvet rolls each partition's Redis stream into segments and offloads sealed segments to the remote tier:

1. Messages are appended to a partition's local Redis stream
2. When a segment reaches its `segment.ms`/`segment.bytes` threshold (10s in this demo) it is sealed
3. The leader-locked storage worker streams sealed segments into the Apache Iceberg table on MinIO, then trims the local tier per the retention settings
4. Reads span both tiers transparently

=== Configuration

[source,yaml]
----
# Local (hot) tier: Redis Enterprise Flex
KORVET_REDIS_URI: redis://host.docker.internal:12000

# Remote (cold) tier: Apache Iceberg table on MinIO (S3-compatible)
KORVET_STORAGE_REMOTE_PATH: s3://korvet/cold
KORVET_STORAGE_REMOTE_S3_ENDPOINT: http://minio:9000
KORVET_STORAGE_REMOTE_S3_PATH_STYLE_ACCESS: "true"
KORVET_STORAGE_REMOTE_S3_ACCESS_KEY_ID: minioadmin
KORVET_STORAGE_REMOTE_S3_SECRET_ACCESS_KEY: minioadmin

# Storage worker tick cadence
KORVET_STORAGE_WORKER_TICK_INTERVAL: 5s
----

== Topic Configuration

Topics must have `remote.storage.enable=true` to be offloaded to the remote tier:

[source,bash]
----
# Create topic with tiered storage
kafka-topics --bootstrap-server localhost:9092 \
  --create --topic my-topic \
  --config remote.storage.enable=true \
  --config local.retention.ms=15000

# Set segment duration so segments seal quickly
kafka-configs --bootstrap-server localhost:9092 \
  --entity-type topics --entity-name my-topic \
  --alter --add-config segment.ms=10000
----

== Monitoring

=== MinIO Console

Access MinIO at http://localhost:9001 (credentials: minioadmin/minioadmin)

=== Redis CLI

[source,bash]
----
make redis-cli

# List local-tier stream segments
KEYS korvet:storage:local:*
----

=== Korvet Metrics

[source,bash]
----
# Storage worker liveness (1 when this instance holds the leader lock)
curl http://localhost:8080/actuator/metrics/korvet.storage.worker.up

# Messages written to the remote tier
curl 'http://localhost:8080/actuator/metrics/korvet.storage.write.messages?tag=tier:remote'
----

== Cleanup

[source,bash]

make down   # Stop services
make clean  # Stop and remove volumes + flush Redis
