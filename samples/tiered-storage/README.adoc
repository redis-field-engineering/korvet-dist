= Korvet Tiered Storage Sample
:toc:
:icons: font

== Overview

This sample demonstrates Korvet's tiered storage feature with:

* **Hot tier**: Redis Enterprise Flex database on `localhost:12000`
* **Cold tier**: MinIO (S3-compatible local storage) with Delta Lake format
* **Bucket duration**: 10 seconds (fast rotation for demo purposes)

Data flows from hot tier to cold tier as buckets age out, enabling cost-effective storage while maintaining Kafka API compatibility.

== Prerequisites

* Docker 20.10+
* Redis Enterprise Flex database running on `localhost:12000`

== Quick Start

[source,bash]
----
cd tiered-storage
make up           # Start Korvet + MinIO
make test-tiered  # Run tiered storage test
make down         # Stop services
----

== How It Works

=== Time-Bucketed Streams

Korvet uses time-bucketed streams for efficient tiering:

1. Messages are written to Redis streams with time-based bucket keys
2. When a bucket's duration expires (10s in this demo), it becomes eligible for archival
3. The archiver moves completed buckets to cold storage (Delta Lake on MinIO)
4. Reads span both tiers transparently

=== Configuration

[source,yaml]
----
# Hot tier: Redis Enterprise Flex
KORVET_REDIS_HOST: host.docker.internal
KORVET_REDIS_PORT: 12000

# Cold tier: MinIO (S3-compatible)
KORVET_STORAGE_REMOTE_PATH: s3a://korvet/delta
KORVET_STORAGE_REMOTE_S3_ENDPOINT: http://minio:9000
KORVET_STORAGE_REMOTE_S3_CREDENTIALS_TYPE: static
KORVET_STORAGE_REMOTE_S3_CREDENTIALS_ACCESS_KEY_ID: minioadmin
KORVET_STORAGE_REMOTE_S3_CREDENTIALS_SECRET_ACCESS_KEY: minioadmin

# Archiver settings
KORVET_STORAGE_REMOTE_ARCHIVER_INTERVAL: 5s
----

== Topic Configuration

Topics must have `remote.storage.enable=true` to be archived:

[source,bash]
----
# Create topic with tiered storage
kafka-topics --bootstrap-server localhost:9092 \
  --create --topic my-topic \
  --config remote.storage.enable=true \
  --config local.retention.ms=15000

# Set bucket duration (Korvet extension)
kafka-configs --bootstrap-server localhost:9092 \
  --entity-type topics --entity-name my-topic \
  --alter --add-config bucket.duration.ms=10000
----

=== Redis CLI

# List stream buckets

----

[source,bash]
----

----
