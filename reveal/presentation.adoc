== Korvet

image::../../korvet.svg[Korvet Logo,100]

*Kafka-on-Redis* (KoR)

[%step]
* Speaks *Kafka wire protocol* → drop-in replacement
* *Redis Streams* for hot, *S3* for cold
* *Best of both worlds:* Redis performance + S3 economics

== Architecture

[source,text]
----
┌──────────┐      ┌──────────┐
│ Filebeat │─────▶│          │
└──────────┘      │          │      ┌─────────────┐
                  │  Korvet  │─────▶│Redis Streams│ (Hot)
┌──────────┐      │  Server  │      └─────────────┘
│ Logstash │─────▶│          │            │
└──────────┘      │ (Kafka   │            │ Tiering
                  │Protocol) │            ▼
┌──────────┐      │          │      ┌────────────┐
│Databricks│◀─────│          │      │   Iceberg  │ (Cold)
│  Spark   │      └──────────┘      │    on S3   │
└──────────┘                        └────────────┘
----

=== Data Flow

[%step]
* *Produce:* Filebeat/Logstash → Kafka protocol → Redis Streams
* *Tier:* Background job moves old data → Delta Lake
* *Consume:* Spark/Databricks → Kafka protocol → Hot or Cold (transparent)
